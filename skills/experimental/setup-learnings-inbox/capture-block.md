## Capturing learnings

This section is opt-in and self-disabling. Act on it **only if**
`docs/learnings/INBOX.md` exists in this repository. If that file is absent,
skip this section entirely — do not create it, and do not record anything.

When the inbox exists: during any task, if you hit something surprising or
non-obvious — a field or status you couldn't trust, an API behaving unlike its
docs, a repo-specific gotcha, or a decision whose rationale wasn't obvious —
append one short entry to `docs/learnings/INBOX.md` before you finish. Capture a
claim plus, where it matters, the *why* or the cost of ignoring it. One entry
per learning, and don't restate things already recorded in
`docs/learnings/LEARNINGS.md`.

Append to the inbox only. Never edit `docs/learnings/LEARNINGS.md` directly —
that file is curated separately. See `docs/learnings/README.md` for the entry
format.
